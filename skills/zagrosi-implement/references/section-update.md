# Section Documentation Update

Before committing a section, update its markdown file so it becomes accurate
implementation documentation rather than stale planning prose.

Update:

- actual files created or modified
- deviations from the original plan and why
- tests added
- code review fixes applied
- operational notes discovered during implementation

Keep it concise and useful. Do not paste large code blocks. Do not change the
section number or name.

If the planning directory is inside the git repo, stage the updated section file
with the section commit. If it is outside the repo, just write the file and note
that the planning docs live separately.
